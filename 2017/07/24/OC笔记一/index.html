OC学习一 基本概念和语法

发表于 2017-07-24 | 分类于 OC

一、OC中的文件
.h文件，函数声明
.m文件，支持OC和C的混编
.mm文件(极少用)，支持OC和C++的混编

二、OC关键字
1、大部分关键字以@开头，因为OC要支持C和C++的混编，不能和C++重名，所以前边加@
2、在xcode中关键字均为紫褐色
3、在自己的代码中不能使用关键字，自己命名的标识符不能使用关键字

三、数据类型
1、所有的数据类型都可以作为函数的参数和函数返回值 ，都可以定义变量
2、OC中有个重要的数据类型NSObject,表示根类
3、d,动态对象类型，可以指向任何对象的指针

四、流程控制语句
和C语言唯一不一样的是，OC中增加了增强for循环，for….in

五、OC的异常处理
和java不一样的是OC的异常处理基本不用

六、import和include区别
import是include的升级版，impoirt自动防止头文件重复包含

七、NSLog和printf比较
1、NSLog是foudation框架中用于日志输出的函数
2、NSLog包含日志对应的日期，以及对应的应用程序
3、自动换行，在末尾加\n是无效的
4、NSLog中的格式化字符串不是普通的C语言字符串，是NSString对象，比如@”abc”字符串就是NSString对象的字面量表示
5、printf中所有的占位符在OC中都是支持的，比如%f，%d，%lf，%c，%s等
6、NSLog新增了格式符%@，用于输出对象

八、结构体和类的区别
C语言中，封装数据用结构体，封装行为用函数，面向对象将二者合二为一，OC中的类，结构体和枚举都应该加上前缀

九、OC中的方法,（重点）

和C语言的写法差别很大，OC方法写法如下：

  • (void) eat: (char*) food using: (char*) tools {

} 

1、方法名为eat: using:，:也是方法名称的一部分   
2、一个 : 对应一个形参   
3、方法声明中的所有数据类型都是用小括号括起来   
4、: 前边的文字描写参数的作用，这样设计增加了方法的可读性    
5、: 前边的参数说明符是可以省略的，但是千万不要省略    
6、最前边加 - 表示方法是对象方法， - 为方法修饰符

十、创建对象

格式：类名 *p = [类名 new]

[类名 new]:创建一个对象

1、为对象在堆中分配内存  
2、返回指向对象的指针  
3、初始化成员变量
